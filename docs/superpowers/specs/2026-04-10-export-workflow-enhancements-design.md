# 导出工作流增强设计

**日期：** 2026-04-10
**范围：** P1 正式可用包、P3 勾选当前存在项、P4 导出前清单预览、P6 推荐勾选按钮

## 目标

在不重做当前 WPF 交互结构的前提下，增强导出页可用性与可确认性：

1. 用户可以一键应用中文命名的推荐勾选（完整迁移 / 轻量常用 / 指标布局）。
2. 用户可以一键勾选当前账号目录中“实际存在文件”的项。
3. 点击“开始导出”前先看到明确的导出预览清单，再确认执行。
4. 提供新的正式单文件发布产物，便于直接使用。
5. 用户可以把当前勾选结果保存到配置1 / 配置2 / 配置3，并在下次一键恢复。

## 已确认的 R1 预设

R1 默认排除以下 4 项：
- `misc_user_root_files`
- `bsdata`
- `user_history_and_cache`
- `limitup_analyse`

其余全部分类均纳入 R1。

## 非目标

- 不重构导入流程。
- 不修改备份包 manifest 格式。
- 不改变核心分类定义，仅改变导出页的选择辅助能力。

## 设计方案

### 1. 推荐勾选（P6）

在导出页新增中文推荐勾选入口：

- 推荐：完整迁移
- 推荐：轻量常用
- 推荐：指标布局

其中：

- 完整迁移：按已确认规则选择大多数有价值配置，仅排除 4 项。
- 轻量常用：轻量常用包。
- 指标布局：极简指标布局包。

推荐预设本身放在独立的、可测试的数据结构中，避免把规则散落在 ViewModel 里。

### 1.1 自定义配置位

在导出页新增 3 个可持久化的自定义配置位：

- 保存为配置1 / 应用配置1
- 保存为配置2 / 应用配置2
- 保存为配置3 / 应用配置3

保存内容为当前勾选的分类 ID 列表，持久化到本地 JSON，供下次直接恢复。

### 2. 勾选当前存在项（P3）

在扫描完成后，界面已掌握每个分类的 `FileCount` 与 `TotalBytes`。新增按钮后，直接基于当前导出页的扫描结果：
- `FileCount > 0` 且可选时设为选中
- `FileCount == 0` 时取消选中

这样不需要重新扫描，也能和当前界面显示保持一致。

### 3. 导出前清单预览（P4）

在真正调用 `BackupPlanner.CreatePlan(...)` 并写 zip 前，先构建导出计划并弹出确认。

预览内容至少包含：
- 已选择分类列表
- 每项文件数 / 大小
- 总文件数 / 总大小
- 导出目标 zip 路径

若用户取消，则不写入 zip，并在日志中记录“已取消导出”。

### 4. 正式可用包（P1）

沿用现有 `scripts/publish-single-file.ps1` 单文件发布链路，增加一个稳定的发布目录/版本化输出说明，并在 README 中补充新功能说明与推荐用法。

## 风险与对策

### 风险 1：预设规则与未来分类变化不一致

对策：
- 用分类 ID 集中定义预设
- 用单元测试锁定 R1 规则

### 风险 2：预览内容与实际导出不一致

对策：
- 预览与导出共享同一个 `BackupPlan`
- 通过测试验证预览统计与导出选择一致

### 风险 3：按钮变多导致 UI 拥挤

对策：
- 先用增量按钮方案实现
- 若视觉过满，再在后续版本做分组或菜单化
